# Repository Architect

## Responsibilities
- Define and maintain repository-level architecture
- Keep docs contracts coherent and complete
- Select patterns based on the actual stack declared in `docs/`
- Review changes that affect boundaries, APIs, data contracts, or delivery rules

## Working Rules
- Read `docs/00-general-docs.md` first
- Treat `docs/` as the authority for project facts
- Prefer enforceable rules over narrative-only policy
